# 问题排查规范
# Claude Code 遇到报错时的标准分析流程

> **核心原则：先定位，后修复。禁止在没有完成系统检查之前直接修改代码。**
>
> 同一个问题修改超过 1 次仍未解决，必须停下来执行本文件的完整排查流程，
> 不得继续盲目尝试。

---

## 第一步：定位问题边界

收到报错后，先回答以下问题，缩小排查范围：

| 问题 | 目的 |
|------|------|
| 报错发生在前端还是后端？ | 确定排查方向 |
| 前端控制台有报错吗？ | 区分网络问题和代码问题 |
| 后端日志有输出吗？ | 判断请求是否到达后端 |
| 这个功能之前能用吗？ | 判断是新引入的问题还是历史问题 |
| 最近改动了哪些文件？ | 缩小改动范围 |

**根据回答确定排查方向：**

```
后端无日志 → 请求未到达后端 → 检查前端网络配置
后端有日志有报错 → 检查后端业务逻辑
前端有报错 → 检查前端代码
前后端都无日志 → 检查代理配置和环境变量
```

---

## 第二步：按层级系统检查

### 前端网络请求问题排查清单

按顺序逐项检查，每项输出实际内容，不得跳过：

```
第一层：环境变量
├── 输出当前启动命令使用的 --mode 参数
├── 输出对应的 .env.[mode] 文件完整内容
├── 确认 VITE_API_BASE_PATH 的值
└── 确认 VITE_USE_MOCK 的值（必须为 false）

第二层：axios 配置
├── 输出 axios 实例配置文件完整内容
├── 确认 baseURL 的取值来源
└── 确认请求拦截器是否正确附加 token

第三层：vite 代理
├── 输出 vite.config.ts 完整内容
├── 确认 proxy 配置的路径前缀
└── 确认 target 指向的后端地址和端口

第四层：后端端口
└── 确认 src/main.ts 中监听的端口与代理 target 一致
```

### 后端业务逻辑问题排查清单

```
第一层：路由
├── 确认请求路径与 controller 的 @Get/@Post 装饰器匹配
└── 确认模块已在 AppModule 中注册

第二层：权限
├── 确认接口是否需要 JWT 验证
├── 确认请求头是否携带了正确的 Authorization
└── 确认角色权限是否匹配

第三层：数据
├── 确认请求参数格式与 DTO 定义一致
├── 确认数据库连接正常
└── 输出具体的错误堆栈信息
```

### Prisma / 数据库问题排查清单

```
第一层：连接
├── 确认 DATABASE_URL 格式正确
├── 确认数据库服务正在运行
└── 执行 npx prisma db pull 验证连接

第二层：Schema
├── 确认字段名使用驼峰格式（非下划线）
├── 确认关联关系双向声明完整
└── 执行 npx prisma validate 检查 schema

第三层：数据
├── 确认查询条件字段存在于表中
└── 确认枚举值与 schema 定义一致
```

---

## 第三步：输出检查结果，再判断原因

完成系统检查后：

1. 把所有检查项的实际值列成表格输出
2. 找出与预期不符的项
3. 分析不符合项之间的因果关系
4. 得出根本原因（通常只有 1-2 个）

**禁止在没有完成以上步骤之前直接修改代码。**

---

## 第四步：修复和验证

1. 针对根本原因制定修复方案，说明修改哪个文件的哪一行
2. 执行修复
3. 用具体操作验证（不能只说"应该好了"）：
   - 网络问题：查看浏览器 Network 面板，确认请求路径和响应码
   - 后端问题：查看 `logs/error.log`，确认无新增错误
   - 数据问题：用验证脚本查询实际数据

---

## 第五步：更新文档

修复完成后，按照 `docs/after-fix-update-docs.md` 更新相关文档。

## 常见问题速查表

| 现象 | 优先检查 |
|------|---------|
| 前后端均无日志，404 | 环境变量 VITE_USE_MOCK、启动命令 --mode |
| 后端无日志，前端有网络错误 | vite 代理配置、后端端口 |
| 后端有日志，返回 401 | JWT token、请求头 Authorization |
| 后端有日志，返回 403 | 角色权限、Guard 配置 |
| 后端有日志，返回 400 | DTO 字段验证、请求参数格式 |
| 后端有日志，返回 1002 | 数据不存在、查询条件错误 |
| Prisma 报字段不存在 | 字段名驼峰格式、schema 与代码不同步 |
| npm install 依赖冲突 | 使用 --legacy-peer-deps |
| prisma generate EPERM | taskkill node.exe、删除 .tmp 文件 |
